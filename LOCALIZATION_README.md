# SoHighPlates 简体中文本地化说明

## 更新内容

本次更新为 SoHighPlates 插件添加了完整的简体中文（zhCN）和繁体中文（zhTW）本地化支持。

### 1. 新增文件

- **locale.lua**: 本地化文件，包含所有 UI 文本的中英文翻译
  - 自动检测客户端语言（GetLocale()）
  - 支持简体中文 (zhCN)、繁体中文 (zhTW) 和英文（默认）
  - 翻译了所有设置选项、滑块标签和对话框文本

### 2. 修改的文件

#### __sNpCore.lua
- **字体配置增强**：
  - 添加了 `isChineseClient` 变量用于检测中文客户端
  - 在 `__sNpCore.art.font` 中添加了 `chinese` 字体配置，使用 WoW 内置的 ARKai_T.ttf 字体
  - 修改了 `ConfigSetValue` 函数中的所有字体设置逻辑
  - 中文客户端会自动使用内置中文字体，避免乱码问题
  - 非中文客户端继续使用原有的自定义字体

#### __sNpPlate.lua
- **姓名板初始化优化**：
  - 在所有文本元素创建时（生命值百分比、等级、姓名、施法时间、法术名称）
  - 添加中文客户端检测，优先使用内置中文字体
  - 确保姓名板上的所有文本都能正确显示中文

#### __sNpOptions.lua
- **UI 本地化**：
  - 导入本地化表 `_G["SoHighPlatesLocals"]`
  - 根据客户端语言选择默认字体（中文客户端使用 ARKai_T.ttf）
  - 所有 UI 文本都使用本地化字符串（L["key"]）
  - 包括：选项标题、复选框标签、滑块标签、按钮文本、确认对话框等

#### SoHighPlates.toc
- **加载顺序调整**：
  - 将 `locale.lua` 添加到文件列表最前面
  - 确保本地化表在其他文件加载前已准备好

### 3. 技术实现细节

#### 字体选择策略
```lua
-- 检测中文客户端
local locale = GetLocale()
local isChineseClient = (locale == "zhCN" or locale == "zhTW")

-- 字体配置
if isChineseClient then
    -- 使用 WoW 内置中文字体
    font = 'Fonts\\ARKai_T.ttf'
else
    -- 使用自定义英文字体
    font = __sNpCore.art.font.cstm.b[1]
end
```

#### 本地化字符串使用
```lua
-- 获取本地化表
local L = _G["SoHighPlatesLocals"] or {}

-- 使用本地化字符串（带回退机制）
_G[widget:GetName()..'Text']:SetText(L["Health Width"] or "Health Width")
```

### 4. 兼容性保证

✅ **多语言兼容**：
- 英文客户端：继续使用原有英文界面和自定义字体
- 中文客户端：自动切换为中文界面和中文字体
- 其他语言客户端：保持英文界面，不受影响

✅ **向后兼容**：
- 现有配置文件完全兼容
- 不影响已有的用户设置
- 自定义字体选项仍然可用（非中文客户端）

✅ **性能优化**：
- 语言检测仅在加载时执行一次
- 使用条件判断避免不必要的字体切换
- 没有额外的运行时开销

### 5. 使用说明

#### 简体中文客户端用户
1. 插件会自动检测您的客户端语言
2. 所有设置选项将显示为中文
3. 姓名板上的姓名、等级、法术名称等文本会正常显示中文
4. 无需任何额外配置

#### 英文/其他语言客户端用户
1. 插件将继续使用英文界面
2. 可以使用自定义字体选项
3. 功能与之前版本完全一致

### 6. 字体说明

**中文客户端使用的字体**：
- `Fonts\ARKai_T.ttf` - WoW 内置楷体（支持简体中文）
- 这是游戏自带的字体文件，无需额外安装
- 适用于所有中文文本显示

**非中文客户端使用的字体**：
- 继续使用插件自带的自定义字体（Angelsk.ttf, Express.ttf 等）
- 这些字体不支持中文，但适合英文显示

⚠️ **重要提示**：
- 如果在中文客户端看到方块或乱码，请确认使用的是 WoW 官方客户端
- 某些私服可能缺少内置中文字体，这种情况下可能需要手动安装字体
- 重置设置后，中文客户端会自动恢复为中文字体
